# Fix: OpenCode Skills Not Showing

## Problem
Skills copied to `~/.config/opencode/skills/` but OpenCode reads commands from `~/.config/opencode/commands/`.

## Solution
Create command files in `~/.config/opencode/commands/` for each skill.

## Command Format (based on existing commands)

```markdown
---
description: [Short description of the skill]
---

[Full skill instructions]
```

## Files to Create

Location: `C:\Users\Colorful\.config\opencode\commands\`

1. `security-audit.md`
2. `bug-investigate.md`
3. `performance-audit.md`
4. `architecture-review.md`
5. `dependency-review.md`
6. `database-review.md`
7. `release-check.md`

## Implementation

For each skill, create a command file that:
1. Has YAML frontmatter with description
2. Contains the skill workflow instructions
3. References shared components

## Verification
1. Restart OpenCode
2. Type `/` to see all commands
3. Test `/security-audit` command
